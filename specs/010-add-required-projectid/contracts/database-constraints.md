# Database Constraints Contract

**Feature**: 010-add-required-projectid
**Date**: 2025-10-03

## Schema Constraints

### Ticket Table

**Foreign Key Constraint**:
```sql
CONSTRAINT "Ticket_projectId_fkey"
  FOREIGN KEY ("projectId")
  REFERENCES "Project"("id")
  ON DELETE CASCADE
  ON UPDATE CASCADE
```

**NOT NULL Constraint**:
```sql
ALTER TABLE "Ticket" ALTER COLUMN "projectId" SET NOT NULL;
```

**Index**:
```sql
CREATE INDEX "Ticket_projectId_idx" ON "Ticket"("projectId");
```

## Constraint Behavior Contracts

### 1. Required Field Constraint
**Contract**: Every Ticket MUST have a projectId value

**Test Scenario**:
```typescript
// MUST FAIL: Attempt to create ticket without projectId
await expect(
  prisma.ticket.create({
    data: {
      title: 'Test Ticket',
      description: 'Test',
      stage: 'INBOX'
      // Missing: projectId
    }
  })
).rejects.toThrow(); // Prisma validation error
```

**Expected Behavior**:
- TypeScript compilation error (missing required field)
- Runtime Prisma error if bypassing type check
- Database rejects INSERT with NULL projectId

### 2. Foreign Key Constraint
**Contract**: projectId MUST reference an existing Project.id

**Test Scenario**:
```typescript
// MUST FAIL: Attempt to create ticket with non-existent projectId
await expect(
  prisma.ticket.create({
    data: {
      title: 'Test Ticket',
      description: 'Test',
      stage: 'INBOX',
      projectId: 99999 // Non-existent project
    }
  })
).rejects.toThrow(); // Foreign key violation
```

**Expected Behavior**:
- Database rejects INSERT with invalid foreign key
- PostgreSQL error code: 23503 (foreign_key_violation)

### 3. Cascade Delete Constraint
**Contract**: Deleting a Project MUST automatically delete all associated Tickets

**Test Scenario**:
```typescript
// Setup: Create project with tickets
const project = await prisma.project.create({
  data: {
    name: 'Test Project',
    description: 'Test',
    githubOwner: 'test',
    githubRepo: 'test',
    tickets: {
      create: [
        { title: 'Ticket 1', description: 'Test', stage: 'INBOX' },
        { title: 'Ticket 2', description: 'Test', stage: 'INBOX' }
      ]
    }
  }
});

// Action: Delete project
await prisma.project.delete({ where: { id: project.id } });

// Verification: All tickets with projectId must be deleted
const orphanedTickets = await prisma.ticket.findMany({
  where: { projectId: project.id }
});

expect(orphanedTickets).toHaveLength(0); // MUST be empty
```

**Expected Behavior**:
- Single DELETE statement on Project triggers cascade
- All Ticket records with matching projectId deleted atomically
- No orphaned tickets remain in database

### 4. Query Performance Constraint
**Contract**: Project-scoped ticket queries MUST use index for performance

**Test Scenario**:
```typescript
// Query tickets by projectId
const tickets = await prisma.ticket.findMany({
  where: { projectId: 1 }
});

// Verify query plan uses index (manual PostgreSQL EXPLAIN check)
```

**Expected Behavior**:
- PostgreSQL query planner uses `Ticket_projectId_idx`
- Query execution time: O(log n) with index
- No full table scan for project filtering

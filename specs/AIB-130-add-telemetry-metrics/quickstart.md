# Quickstart: Add Telemetry Metrics to Ticket Comparison

## Implementation Overview

This feature adds telemetry metrics (cost, duration, tokens) to the `/compare` command by:
1. **Workflow**: Fetching job telemetry via API before Claude executes
2. **Context File**: Writing aggregated data to `.telemetry-context.json`
3. **Compare Command**: Reading context file to populate metrics table

## Key Files to Modify

### 1. Workflow: `.github/workflows/ai-board-assist.yml`

**Add step before "Execute AI-BOARD Command"**:

```yaml
- name: Fetch Telemetry for Compare
  if: ${{ env.SKIP_CLAUDE != 'true' && contains(inputs.comment, '/compare') }}
  env:
    BRANCH: ${{ inputs.branch }}
    PROJECT_ID: ${{ inputs.project_id }}
  run: |
    # Extract ticket references from comment
    TICKETS=$(echo "${{ inputs.comment }}" | grep -oE '#[A-Z0-9]{3,6}-[0-9]+' | tr -d '#' | sort -u)

    # Initialize telemetry JSON
    TELEMETRY_JSON="{\"generatedAt\": \"$(date -u +%Y-%m-%dT%H:%M:%SZ)\", \"tickets\": {}}"

    for TICKET_KEY in $TICKETS; do
      echo "📊 Fetching telemetry for $TICKET_KEY..."

      # Resolve ticket key to ID
      SEARCH_RESULT=$(curl -s -H "Authorization: Bearer ${WORKFLOW_API_TOKEN}" \
        "${APP_URL}/api/projects/${PROJECT_ID}/tickets/search?q=${TICKET_KEY}" || echo '{"results":[]}')

      TICKET_ID=$(echo "$SEARCH_RESULT" | jq -r '.results[0].id // empty')

      if [ -z "$TICKET_ID" ]; then
        echo "⚠️ Ticket $TICKET_KEY not found, using empty telemetry"
        TELEMETRY=$(jq -n --arg key "$TICKET_KEY" '{
          ticketKey: $key, inputTokens: 0, outputTokens: 0, cacheReadTokens: 0,
          cacheCreationTokens: 0, costUsd: 0, durationMs: 0, model: null,
          toolsUsed: [], jobCount: 0, hasData: false
        }')
      else
        # Fetch jobs and aggregate
        JOBS=$(curl -s -H "Authorization: Bearer ${WORKFLOW_API_TOKEN}" \
          "${APP_URL}/api/projects/${PROJECT_ID}/tickets/${TICKET_ID}/jobs" || echo '[]')

        TELEMETRY=$(echo "$JOBS" | jq --arg key "$TICKET_KEY" '{
          ticketKey: $key,
          inputTokens: ([.[] | select(.status == "COMPLETED") | .inputTokens // 0] | add) // 0,
          outputTokens: ([.[] | select(.status == "COMPLETED") | .outputTokens // 0] | add) // 0,
          cacheReadTokens: ([.[] | select(.status == "COMPLETED") | .cacheReadTokens // 0] | add) // 0,
          cacheCreationTokens: ([.[] | select(.status == "COMPLETED") | .cacheCreationTokens // 0] | add) // 0,
          costUsd: ([.[] | select(.status == "COMPLETED") | .costUsd // 0] | add) // 0,
          durationMs: ([.[] | select(.status == "COMPLETED") | .durationMs // 0] | add) // 0,
          model: ([.[] | select(.status == "COMPLETED" and .model != null) | .model] | first) // null,
          toolsUsed: ([.[] | select(.status == "COMPLETED") | .toolsUsed[]] | unique | sort) // [],
          jobCount: ([.[] | select(.status == "COMPLETED")] | length) // 0,
          hasData: (([.[] | select(.status == "COMPLETED") | .costUsd // 0] | add) // 0) > 0
        }')
      fi

      # Add to combined JSON
      TELEMETRY_JSON=$(echo "$TELEMETRY_JSON" | jq --arg key "$TICKET_KEY" --argjson tel "$TELEMETRY" '.tickets[$key] = $tel')
    done

    # Write context file
    mkdir -p "specs/${BRANCH}"
    echo "$TELEMETRY_JSON" > "specs/${BRANCH}/.telemetry-context.json"
    echo "✅ Telemetry context written to specs/${BRANCH}/.telemetry-context.json"
```

### 2. Compare Command: `.claude/commands/compare.md`

**Add to Step 6 (Aggregate Telemetry)**:

```markdown
### Step 6: Aggregate Telemetry

**Read the telemetry context file generated by the workflow**:

\`\`\`bash
cat specs/$BRANCH/.telemetry-context.json
\`\`\`

This file contains pre-aggregated telemetry for all referenced tickets. Parse the JSON:

- `tickets[ticketKey].hasData` - Whether telemetry is available
- `tickets[ticketKey].costUsd` - Total cost in USD
- `tickets[ticketKey].durationMs` - Total duration in milliseconds
- `tickets[ticketKey].inputTokens` - Input tokens consumed
- `tickets[ticketKey].outputTokens` - Output tokens generated

**For tickets with `hasData: false`**, display "N/A" in the metrics table.
```

**Update Metrics Comparison table format**:

```markdown
## Metrics Comparison
| Ticket | Lines | Files | Tests | Cost | Duration |

| AIB-127 | 120 | 4 | 2 | $0.0892 | 28s |
| AIB-129 | N/A | N/A | N/A | N/A | N/A |
```

## Testing Approach

### Integration Test

Create `tests/integration/telemetry/workflow-context.test.ts`:

```typescript
import { describe, it, expect } from 'vitest';

describe('Telemetry Context File', () => {
  it('should match TicketTelemetry schema', async () => {
    const context = {
      generatedAt: new Date().toISOString(),
      tickets: {
        'AIB-127': {
          ticketKey: 'AIB-127',
          inputTokens: 35000,
          outputTokens: 18000,
          cacheReadTokens: 8000,
          cacheCreationTokens: 3000,
          costUsd: 0.1023,
          durationMs: 33000,
          model: 'claude-sonnet-4-5-20251101',
          toolsUsed: ['Edit', 'Read'],
          jobCount: 2,
          hasData: true,
        },
      },
    };

    // Validate structure matches TicketTelemetry
    const telemetry = context.tickets['AIB-127'];
    expect(telemetry.ticketKey).toBe('AIB-127');
    expect(typeof telemetry.costUsd).toBe('number');
    expect(typeof telemetry.hasData).toBe('boolean');
  });
});
```

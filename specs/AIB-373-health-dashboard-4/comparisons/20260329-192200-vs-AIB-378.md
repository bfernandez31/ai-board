## Implementation Choices Analysis

### [Spec] FR-001: Schema-Compatible JSON Output
| Ticket | Approach |
|--------|----------|
| AIB-373 | Includes `type` discriminator field (`SECURITY`, `COMPLIANCE`, `TESTS`, `SPEC_SYNC`), `generatedTickets: []`, issue `id` fields — matches `scanReportSchema` discriminated union exactly |
| AIB-378 | Omits `type` field (relies on `parseScanReport` fallback injection), omits `generatedTickets`, omits issue `id` — would fail `reportIssueSchema` validation on missing `id` |

**Best**: AIB-373 — Output passes the existing Zod schema in `lib/health/report-schemas.ts` without fallback hacks.

**Best**: AIB-373 — Matches the existing Zod schema structure for TESTS reports.

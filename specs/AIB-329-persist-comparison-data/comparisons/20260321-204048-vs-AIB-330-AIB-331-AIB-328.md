**Best**: AIB-331 — Correct ticket-level scoping with source ticket ID/key match validation.

### Decision Point 2: Zod Validation Strategy
| Ticket | Approach | Constitution | Trade-offs |
|--------|----------|--------------|------------|
| AIB-329 | `z.enum([...])` string literals | Yes | Explicit but duplicates Prisma enums |

| AIB-328 | `z.looseObject({})` for nested | Weak | Loose validation defeats purpose |

**Best**: AIB-331 — Uses `z.nativeEnum` for Prisma enum types, ensuring validation stays in sync with schema.

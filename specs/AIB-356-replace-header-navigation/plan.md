# Implementation Plan: Replace Header Navigation with Icon Rail Sidebar + Command Palette

**Branch**: `AIB-356-replace-header-navigation` | **Date**: 2026-03-26 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/AIB-356-replace-header-navigation/spec.md`

## Summary

Replace the header's navigation icon buttons (Specs, Analytics, Activity) with a 48px-wide vertical icon rail sidebar on project pages (вүҘ1024px viewport) and add a `Cmd+K`/`Ctrl+K` command palette for unified search and navigation. The header is simplified to: project name, search trigger with keyboard shortcut badge, notification bell, and user avatar. Mobile navigation via hamburger menu remains unchanged. No database changes вҖ” purely frontend UI refactor using shadcn/ui Command component (cmdk) and CSS Grid layout.

## Technical Context

**Language/Version**: TypeScript 5.6 (strict), React 18, Next.js 16 (App Router)
**Primary Dependencies**: shadcn/ui, cmdk (new), lucide-react, TanStack Query v5, @radix-ui/react-dialog (existing)
**Storage**: N/A (no database changes)
**Testing**: Vitest (unit + component + integration), Playwright (E2E if browser-required)
**Target Platform**: Web (desktop вүҘ1024px with icon rail, mobile <1024px without)
**Project Type**: Web application (Next.js App Router)
**Performance Goals**: Command palette opens in <100ms, navigation result filtering instant (client-side), ticket search <500ms (existing API)
**Constraints**: 48px sidebar must not reduce usable board area below 6-column threshold at 1280px
**Scale/Scope**: 4 navigation items, reuses existing ticket search API

### Icon Rail Sidebar (`components/navigation/icon-rail-sidebar.tsx`)

- **Width**: Fixed 48px, rendered via CSS Grid column
- **Visibility**: `hidden lg:flex` (вүҘ1024px only)
- **Active detection**: `usePathname()` compared against navigation item hrefs
- **Active indicator**: Background highlight on active icon (e.g., `bg-accent rounded-md`)
- **Tooltips**: shadcn/ui `Tooltip` on each icon (hover and keyboard focus)
- **Layout**: Flex column with `justify-between` вҖ” views group top, settings bottom
- **Divider**: Subtle `border-t` between views group and settings
- **Accessibility**: `nav` landmark with `aria-label="Project navigation"`, each link has `aria-current="page"` when active

### Command Palette (`components/navigation/command-palette.tsx`)

- **Trigger**: `Cmd+K` (Mac) / `Ctrl+K` (Windows/Linux) global keydown listener
- **Also triggered by**: SearchTrigger click, S/slash shortcut from board
- **UI**: shadcn/ui `CommandDialog` (Command + Dialog wrapper)
- **Groups**:
  1. "Navigation" вҖ” static list filtered by cmdk built-in fuzzy match
  2. "Tickets" вҖ” async results from `useTicketSearch` with loading state
- **Keyboard navigation**: Built-in via cmdk (arrow keys, Enter, Escape)
- **Empty state**: "No results found" message via `CommandEmpty`
- **Modal stacking prevention**: Before opening, check `document.querySelector('[role="dialog"][data-state="open"]')`
- **On select**: Navigate via `router.push()`, close palette
- **State reset**: Each open starts with empty search input
- **Same-page navigation**: Close gracefully without redundant navigation (compare current pathname)

### Search Trigger (`components/navigation/search-trigger.tsx`)

- **Appearance**: Styled like an input field (border, muted text) but is a `<button>`
- **Content**: Search icon + "Search..." placeholder + `вҢҳK` / `Ctrl+K` badge
- **Badge**: Detects Mac via `navigator.platform` or `navigator.userAgent` for correct modifier display
- **Click handler**: Opens command palette
- **Visibility**: Desktop only (hidden on mobile), shown on all project pages (not just board)

### Project Layout (`app/projects/[projectId]/layout.tsx`)

- **Server Component** that renders the grid structure
- **CSS Grid**: `lg:grid lg:grid-cols-[48px_1fr]` with full-width fallback on mobile
- **Contains**: IconRailSidebar (client), children (pages), CommandPalette (client)
- **Command palette state**: Managed here, passed to both SearchTrigger (via header) and CommandPalette
- **Note**: CommandPalette needs `projectId` from route params вҖ” available in layout via `params`

### E2E Tests вҖ” Deferred

E2E tests are not planned for initial implementation. The command palette and icon rail can be fully tested via component and integration tests using Vitest + RTL. E2E tests may be added during the VERIFY stage if browser-specific issues are discovered (e.g., viewport-dependent rendering, focus management across modals).

**Rationale**: Per constitution and CLAUDE.md, E2E is expensive (~5s each) and should only be used for browser-required features. The command palette's keyboard handling and focus management can be simulated with `userEvent` in RTL. CSS Grid responsive behavior can be tested via class assertions.

## Summary

Enhance the GitHub Actions workflow for the `/speckit.implement` command to automatically set up a complete test environment (PostgreSQL database + Playwright) and provide Claude with explicit instructions to run only impacted tests during implementation. This enables autonomous implementation validation without manual intervention while optimizing test execution time.

**Testing**: Playwright (E2E/API tests), Vitest (unit tests), global-setup.ts (test data seeding)
**Target Platform**: GitHub Actions ubuntu-latest runner (cloud CI/CD)
**Project Type**: Web application (Next.js frontend + API routes backend)
**Performance Goals**: Database setup <2 minutes, Playwright installation <3 minutes (with caching), selective test execution reduces runtime by 50%+, total workflow <10 minutes
**Constraints**: GitHub Actions timeout 120 minutes, must work in ephemeral CI environment, zero manual intervention required
**Scale/Scope**: Single workflow file modification (.github/workflows/speckit.yml), affects implement command only, must maintain backward compatibility with specify/plan commands

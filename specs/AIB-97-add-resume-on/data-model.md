# Data Model: Implementation Summary Output

**Feature**: AIB-97-add-resume-on
**Date**: 2025-12-05

## Overview

This feature operates exclusively with file-based artifacts. No database entities are involved.

## File Entities

### Summary Template

**Location**: `.specify/templates/summary-template.md`
**Type**: Static template file
**Purpose**: Defines the structure for generated summaries

**Structure**:
```markdown
# Implementation Summary: [FEATURE_NAME]

**Branch**: `[BRANCH]` | **Date**: [DATE]
**Spec**: [link to spec.md]

## Changes Summary

[Brief description of what was implemented - max 500 chars]

## Key Decisions

[Important technical decisions made during implementation - max 500 chars]

## Files Modified

[List of key files created/modified - max 500 chars]

## ⚠️ Manual Requirements

[Any steps requiring human action, or "None" if fully automated - max 300 chars]
```

### Generated Summary

**Location**: `specs/[feature]/summary.md`
**Type**: Generated output file
**Purpose**: Records implementation outcomes for the specific feature
**Constraints**:
- Maximum 2300 characters total
- Generated after `/speckit.implement` completion
- Follows template structure

## Validation Rules

| Field | Rule |
|-------|------|
| FEATURE_NAME | Extracted from spec.md header |
| BRANCH | Current git branch |
| DATE | ISO 8601 date (YYYY-MM-DD) |
| Total content | ≤ 2300 characters |

## Relationships

```
.specify/templates/summary-template.md
         │
         │ (provides structure for)
         ▼
specs/[feature]/summary.md ◄─── generated by /speckit.implement
         │
         │ (links to)
         ▼
specs/[feature]/spec.md
```

## No Database Entities

This feature does not modify the Prisma schema or require database migrations.

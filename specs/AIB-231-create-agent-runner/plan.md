## Summary

Create a unified shell script (`run-agent.sh`) that abstracts CLI installation, authentication, telemetry configuration, and command invocation across Claude Code and Codex CLI agents. Update all 6 GitHub workflow files to use this script instead of hardcoded Claude-specific steps, enabling multi-agent support while maintaining zero regression for existing Claude workflows.

## Design Decisions

### D-001: Script Architecture

`run-agent.sh` uses a function-per-agent pattern:

```bash
#!/usr/bin/env bash
set -euo pipefail

AGENT_TYPE="${1:?}"
COMMAND="${2:?}"
shift 2
ARGS="$*"

# Validation
validate_auth() { ... }

# Installation
install_claude() { bun add -g @anthropic-ai/claude-code; }
install_codex()  { bun add -g @openai/codex && codex login --api-key "$OPENAI_API_KEY"; }

# Environment setup
setup_codex_env() {
  # Generate AGENTS.md from CLAUDE.md
  # Write ~/.codex/config.toml for telemetry
}

# Invocation
invoke_claude() { claude --dangerously-skip-permissions "/$COMMAND $ARGS"; }
invoke_codex()  {
  # Read .claude/commands/${COMMAND}.md
  # Concatenate with ARGS
  # Pipe to: codex exec --full-auto -m "$CODEX_MODEL" -
}

# Main dispatch
case "$AGENT_TYPE" in
  CLAUDE) validate_auth; install_claude; invoke_claude ;;
  CODEX)  validate_auth; install_codex; setup_codex_env; invoke_codex ;;
  *)      echo "ERROR: Unsupported agent type '$AGENT_TYPE'"; exit 1 ;;
esac
```

### D-002: Workflow Modification Strategy

Each workflow modification follows the same pattern:
1. **Remove** the "Install Claude Code CLI" step
2. **Replace** each `claude --dangerously-skip-permissions` invocation with `run-agent.sh` call
3. **Preserve** all environment variables in the workflow env block (telemetry, model)
4. **Add** `agent` input to quick-impl.yml (the only workflow missing it)

The `run-agent.sh` script reads telemetry env vars from the environment (set by workflow), so no telemetry logic needs to move into the script for the Claude path.

### D-003: Codex Telemetry Mapping

Codex uses `~/.codex/config.toml` instead of environment variables. The script translates:

| Workflow Env Var | Codex config.toml field |
|-----------------|------------------------|
| `OTEL_EXPORTER_OTLP_ENDPOINT` | `otel.exporter.otlp-http.endpoint` |
| `OTEL_EXPORTER_OTLP_HEADERS` | `otel.exporter.otlp-http.headers` |
| `OTEL_EXPORTER_OTLP_PROTOCOL` | `otel.exporter.otlp-http.protocol` |

### D-004: AGENTS.md Generation

For Codex, the script:
1. Checks if `CLAUDE.md` exists in the working directory
2. Copies content to `AGENTS.md`
3. Checks file size; if > 32KB, truncates at the 32KB boundary with a truncation notice
4. Does NOT commit the file (ephemeral CI artifact)

### D-005: Command .md File Resolution

For Codex, the command identifier `ai-board.specify` maps to:
- File path: `.claude/commands/ai-board.specify.md`
- This path works because all workflows already create the `.claude/commands` symlink before calling `run-agent.sh`

The .md file content is concatenated with the arguments and piped to `codex exec` via stdin to avoid shell argument length limits.

# Implementation Plan: Modernize Comparison Dashboard Visual Design

**Branch**: `AIB-361-modernize-comparison-dashboard` | **Date**: 2026-03-27 | **Spec**: `specs/AIB-361-modernize-comparison-dashboard/spec.md`
**Input**: Feature specification from `/specs/AIB-361-modernize-comparison-dashboard/spec.md`

## Summary

Modernize the comparison dashboard's visual design by applying glassmorphism styling, rank-based accent color identity per participant, SVG gradient score gauges with glow effects, color-themed stat cards, enhanced decision point cards, gradient metric bars with color legends, and colored compliance heatmap cells. All changes are purely visual — no component hierarchy, data flow, or prop changes. All colors use existing Catppuccin palette tokens via Tailwind semantic classes.

## Implementation Approach

### Accent Color System

A single utility (`lib/comparison/accent-colors.ts`) maps rank positions to Catppuccin color tokens:

| Rank | Color Name | Tailwind Token | Use Cases |
|------|-----------|----------------|-----------|
| 1 (Winner) | Green | `ctp-green` | Hero card bg, score gauge, badges, metrics bars |
| 2 | Blue | `ctp-blue` | Participant card, gauge, bars, decision verdicts |
| 3 | Mauve | `ctp-mauve` | Same as above |
| 4 | Peach | `ctp-peach` | Same as above |
| 5 | Pink | `ctp-pink` | Same as above |
| 6 | Yellow | `ctp-yellow` | Same as above |

The utility returns complete class sets per rank: `{ text, bg, border, bgSubtle, ring }` — all as complete static Tailwind class strings (never dynamically constructed).

### Glassmorphism Base

All comparison cards receive:
- Background: `bg-ctp-surface0/[0.04]` (3-4% opacity)
- Border: `border border-ctp-overlay0/10` (8-10% opacity)
- Increased padding: `p-6` (up from `p-4`)
- Increased gaps: `gap-6` between sections

### Component Modifications (high-level)

1. **ScoreGauge**: Add `<defs>` with `<linearGradient>` using accent color stops; add `<filter>` for drop-shadow glow; apply gradient as stroke URL
2. **HeroCard**: Gradient background via accent color at 12% opacity; radial glow orb (absolute positioned div); gradient winner badge; bordered recommendation container; colored differentiator pills
3. **ParticipantGrid**: Per-card accent color tint, border, rank badge color, mini score ring color
4. **StatCards**: Per-card color themes (yellow/blue/green/purple) for label, bar gradient, background, border
5. **DecisionPoints**: Individual card styling with verdict winner's accent color background tint; glowing dot with colored box-shadow; verdict pill badge
6. **UnifiedMetrics**: Color legend header; gradient bars per participant; bold + accent color for winner values
7. **ComplianceHeatmap**: Subtle colored backgrounds already partially implemented — enhance with consistent opacity levels

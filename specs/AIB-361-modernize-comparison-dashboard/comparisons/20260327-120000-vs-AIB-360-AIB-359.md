## Implementation Choices Analysis

### [Spec] Score values 18px font-weight 800
| Ticket | Approach |
|--------|----------|
| AIB-361 | `text-lg font-extrabold tracking-tight` (close but not exact 18px) |
| AIB-360 | `text-[18px] font-extrabold tracking-[-0.05em]` (exact spec match) |
| AIB-359 | `text-sm font-bold` (does not meet spec) |

**Best**: AIB-360 - Exact match for FR-013.

### [Spec] No structural/layout changes (FR-028)
| Ticket | Approach |
|--------|----------|
| AIB-361 | Wraps existing components in section elements with glassmorphism classes; minimal structural impact |
| AIB-360 | Restructures unified-metrics from table to card layout; removes Card wrappers from compliance/decision-points |
| AIB-359 | Removes Card/CardContent from compliance, decision-points, hero-card; replaces with plain divs |

**Best**: AIB-361 - Least invasive; existing component hierarchy preserved.

### [Spec] Hardcoded colors (FR-025)
| Ticket | Approach |
|--------|----------|
| AIB-361 | Pure Tailwind tokens + HSL CSS vars in SVG defs only |
| AIB-360 | Mostly Tailwind tokens; uses `border-white/10` extensively and HSL vars in SVG defs/filters |
| AIB-359 | Inline `style={{ boxShadow }}` and `style={{ background: 'radial-gradient(...)' }}` with CSS vars |

**Best**: AIB-361 - Cleanest Tailwind-only approach.

### Decision Point 1: Color System Architecture
| Ticket | Approach | Constitution | Trade-offs |
|--------|----------|--------------|------------|
| AIB-361 | `lib/comparison/accent-colors.ts` — separate utility with dedicated unit tests | Yes (II: shared utilities in `/lib/`) | Clean SRP, testable, 91 lines |
| AIB-360 | `components/comparison/comparison-theme.ts` — comprehensive 170-line theme with 12+ properties per rank | Partially (file in components/ not lib/) | Most thorough but heavier abstraction |
| AIB-359 | `components/comparison/participant-colors.ts` — color + `formatEnrichmentValue` utility mixed | No (SRP violation: formatting utility mixed with color definitions) | 133 lines, includes unrelated utility |

**Best**: AIB-361 - Proper lib/ placement per constitution, SRP, dedicated tests.

### Decision Point 2: SVG Gradient & Glow Implementation
| Ticket | Approach | Constitution | Trade-offs |
|--------|----------|--------------|------------|
| AIB-361 | Passes `accentColor` HSL string to ScoreGauge; gauge creates linearGradient internally | Yes | Simple API, 1 new prop |
| AIB-360 | Passes full `ComparisonAccentTheme` object + `gradientId`; gauge creates gradient + feDropShadow filter | Yes | Rich effects (glow filter), 2 new props, more complex |
| AIB-359 | Passes `strokeColor` + `glow` CSS string; gauge uses linearGradient + drop-shadow via CSS filter | Yes | 2 new props, inline CSS for glow |

**Best**: AIB-360 - SVG filter-based glow is more precise than CSS drop-shadow; however AIB-361's simpler API is more maintainable.

### Decision Point 3: Test Coverage Strategy
| Ticket | Approach | Constitution | Trade-offs |

| AIB-359 | 4 test files updated only | Partial (missing tests for 5 components) | Weakest coverage |

**Best**: AIB-361 - Most comprehensive test coverage with dedicated utility tests.

## Ranking & Recommendation

### 1. **AIB-361** - Best code (score: 91%)
**Why it wins**:
- Cleanest architecture: accent-colors utility properly placed in `lib/` with dedicated unit tests
- Best test coverage: 8 test files, 89% test ratio
- Least invasive structural changes — preserves component hierarchy best
- Pure Tailwind tokens — no inline style workarounds
- Already updated consolidated specs

### 2. **AIB-360** - Strong runner-up (score: 88%)
**Why it lost**:
- Comprehensive 170-line theme system may be over-engineered for this feature
- Theme file placed in `components/` instead of `lib/` per constitution
- Uses `border-white/10` (not semantic token)
- Restructured unified-metrics layout (violates FR-028)
- Lowest cost ($5.16) demonstrates efficiency

### 3. **AIB-359** - (score: 76%)
**Why it lost**:
- Only 4 test files (44% ratio) — weakest test coverage
- SRP violation: `formatEnrichmentValue` mixed into color utility
- Removes Card/CardContent wrappers (structural changes violating FR-028)
- Inline styles for glow effects (`boxShadow`, `background`)

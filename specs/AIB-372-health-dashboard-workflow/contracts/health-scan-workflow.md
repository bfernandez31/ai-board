# Contract: health-scan.yml Workflow

**Type**: GitHub Actions Workflow (workflow_dispatch)
**Location**: `.github/workflows/health-scan.yml`

## Workflow Inputs

| Input | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `scan_id` | string | yes | — | HealthScan record ID |
| `project_id` | string | yes | — | Project ID for API calls |
| `scan_type` | string | yes | — | One of: SECURITY, COMPLIANCE, TESTS, SPEC_SYNC |
| `base_commit` | string | no | `''` | Previous scan's headCommit (empty = full scan) |
| `head_commit` | string | no | `''` | Target commit (empty = use HEAD after checkout) |
| `githubRepository` | string | yes | — | Target repo in `owner/repo` format |
| `agent` | string | no | `CLAUDE` | Agent CLI: CLAUDE or CODEX |

## Secrets Required

| Secret | Purpose |
|--------|---------|
| `WORKFLOW_API_TOKEN` | Bearer token for ai-board API calls (status updates, ticket creation) |
| `GH_PAT` | GitHub token for cross-repo checkout |
| `CLAUDE_CODE_OAUTH_TOKEN` | Claude Code CLI authentication |
| `CODEX_AUTH_JSON` | Codex CLI authentication (when agent=CODEX) |

## Environment Variables

| Variable | Source | Purpose |
|----------|--------|---------|
| `APP_URL` | `vars.APP_URL` | ai-board API base URL |
| `ANTHROPIC_MODEL` | hardcoded | `claude-opus-4-6` |
| `CLAUDE_CODE_ENABLE_TELEMETRY` | hardcoded | `1` |
| `OTEL_*` | configured | Telemetry export to ai-board OTEL endpoint |

## API Calls Made by Workflow

### 1. Update Scan Status to RUNNING
```
PATCH {APP_URL}/api/projects/{project_id}/health/scans/{scan_id}/status
Authorization: Bearer {WORKFLOW_API_TOKEN}
Content-Type: application/json

{ "status": "RUNNING" }
```

### 2. Update Scan Status to COMPLETED
```
PATCH {APP_URL}/api/projects/{project_id}/health/scans/{scan_id}/status
Authorization: Bearer {WORKFLOW_API_TOKEN}
Content-Type: application/json

{
  "status": "COMPLETED",
  "score": 85,
  "report": "{...JSON...}",
  "issuesFound": 12,
  "issuesFixed": 3,
  "headCommit": "abc123...40chars",
  "durationMs": 45000,
  "tokensUsed": 150000,
  "costUsd": 1.23
}
```

### 3. Update Scan Status to FAILED
```
PATCH {APP_URL}/api/projects/{project_id}/health/scans/{scan_id}/status
Authorization: Bearer {WORKFLOW_API_TOKEN}
Content-Type: application/json

{
  "status": "FAILED",
  "errorMessage": "Scan command failed: exit code 1",
  "durationMs": 5000
}
```

### 4. Create Ticket (per issue group)
```
POST {APP_URL}/api/projects/{project_id}/tickets
Authorization: Bearer {WORKFLOW_API_TOKEN}
Content-Type: application/json

{
  "title": "[Health:Security] High severity issues",
  "description": "## Security Scan Issues (High Severity)\n\nScan ID: 42\n\n- **SQL Injection** in `src/api/users.ts:15`\n- **XSS Vulnerability** in `src/components/Form.tsx:88`\n\n*Auto-generated by health scan*"
}
```

## Scan Type → Command Mapping

| scan_type | Command Name | Report Schema |
|-----------|-------------|---------------|
| `SECURITY` | `health-security` | SecurityReport |
| `COMPLIANCE` | `health-compliance` | ComplianceReport |
| `TESTS` | `health-tests` | TestsReport |
| `SPEC_SYNC` | `health-spec-sync` | SpecSyncReport |

## Command Arguments

Commands receive arguments as a string:
```
run-agent.sh {AGENT} {COMMAND} "scan_id={scan_id} project_id={project_id} base_commit={base_commit} head_commit={head_commit}"
```

## Command Output Contract

Commands MUST produce valid JSON to stdout matching the corresponding report schema from `lib/health/report-schemas.ts`. The `generatedTickets` array should be empty (workflow populates it after ticket creation).

## Ticket Grouping Rules

| Scan Type | Group By | Ticket Title Pattern | Ticket Count |
|-----------|----------|---------------------|--------------|
| SECURITY | `severity` (high/medium/low) | `[Health:Security] {Severity} severity issues` | 0-3 |
| COMPLIANCE | `category` (principle name) | `[Health:Compliance] {Principle} violations` | 0-N |
| TESTS | Each `nonFixable` item | `[Health:Tests] {test description}` | 0-N |
| SPEC_SYNC | Each drifted spec | `[Health:SpecSync] {specPath} drift` | 0-N |

## Execution Flow

```
┌─────────────────────────────────────────────────────────────┐
│ 1. Receive workflow_dispatch with inputs                     │
│ 2. PATCH status → RUNNING                                    │
│ 3. Sparse checkout ai-board (commands + scripts)             │
│ 4. Full checkout target repo (fetch-depth: 0)                │
│ 5. Symlink commands: target/.claude/commands → ai-board/...  │
│ 6. Capture HEAD SHA                                          │
│ 7. Validate base_commit (fallback to full scan if invalid)   │
│ 8. Execute: run-agent.sh {agent} {command} {args}            │
│ 9. Parse JSON report from stdout                             │
│ 10. Create grouped tickets via POST /tickets                 │
│ 11. Record ticket keys in report.generatedTickets            │
│ 12. PATCH status → COMPLETED (score, report, telemetry)      │
│                                                               │
│ On error at any step:                                         │
│ → PATCH status → FAILED (errorMessage, durationMs)           │
└─────────────────────────────────────────────────────────────┘
```

## Error Scenarios

| Error | Behavior |
|-------|----------|
| Repo checkout fails | FAILED with "Repository access failed" |
| base_commit not found | Fallback to full scan (base_commit = empty) |
| Scan command exits non-zero | FAILED with command error output |
| Report JSON invalid | FAILED with "Invalid scan report format" |
| Ticket creation fails | Log warning, continue remaining tickets, scan still COMPLETED |
| Status update fails | Log warning (workflow continues; scan stuck in previous state) |

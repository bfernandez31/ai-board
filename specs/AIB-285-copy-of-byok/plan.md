# Implementation Plan: BYOK - Bring Your Own API Key

**Branch**: `AIB-285-copy-of-byok` | **Date**: 2026-03-13 | **Spec**: `specs/AIB-285-copy-of-byok/spec.md`
**Input**: Feature specification from `/specs/AIB-285-copy-of-byok/spec.md`

## Summary

Enable project owners to configure their own Anthropic and OpenAI API keys so AI workflow costs are borne by users. Keys are encrypted at rest (AES-256-GCM), validated against provider APIs before saving, and automatically injected into workflow dispatch inputs. Existing projects without BYOK keys continue using repository-level secrets with zero disruption.

## Technical Context

**Language/Version**: TypeScript 5.6 (strict), Node.js 22.20.0
**Primary Dependencies**: Next.js 16 (App Router), React 18, Prisma 6.x, shadcn/ui, TanStack Query v5, Node.js `crypto` (built-in)
**Storage**: PostgreSQL 14+ via Prisma ORM (new `ProjectAPIKey` model)
**Testing**: Vitest (unit + integration), Playwright (E2E browser tests only)
**Target Platform**: Vercel (Next.js deployment)
**Project Type**: Web application (Next.js App Router)
**Performance Goals**: Key validation response within 5 seconds (SC-007); encrypt/decrypt overhead < 10ms
**Constraints**: No external encryption libraries; use Node.js built-in `crypto`. No plaintext keys in DB, logs, or API responses.
**Scale/Scope**: One key per provider per project; two providers (Anthropic, OpenAI)

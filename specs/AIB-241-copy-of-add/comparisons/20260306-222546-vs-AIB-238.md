### Decision Point 2: Preserving Component-Level delayDuration
| Ticket | Approach | Correct |
|--------|----------|---------|
| AIB-241 | Moves `delayDuration={300}` from TooltipProvider to `<Tooltip>` in MentionChip | Yes |
| AIB-238 | Drops `delayDuration={300}` entirely | **No - Bug** |

**Best**: AIB-241. The original MentionChip used `<TooltipProvider delayDuration={300}>` to customize tooltip delay. AIB-241 correctly migrates this to `<Tooltip delayDuration={300}>`. AIB-238 silently drops it, reverting to the default delay. This is a **functional regression**.

### Decision Point 3: Test Infrastructure Updates
| Ticket | Approach | Correct |

| AIB-241 | No test util changes | Missing |

**Best**: AIB-238. Wrapping test renders with TooltipProvider ensures component tests don't break when local providers are removed. AIB-241 misses this.

### Decision Point 4: Deploy Icon Tooltip Text Logic
| Ticket | Approach | Notes |
